# 멤버 조성민의 작업 내용 상세 보고

---

## 주요 기여 내용 요약

- **플레이어 및 AI 캐릭터 베이스 시스템 구축**: GAS (`UAbilitySystemComponent`, `UAttributeSet`) 기반으로 캐릭터의 스탯, 어빌리티, 팀 정보를 통합 관리하는 베이스 클래스를 설계했습니다.
- **데이터 기반 캐릭터 구조 설계**: `PlayerDataAsset`을 통해 캐릭터의 초기 스탯, 스킬, 입력을 자동 세팅하도록 구현했습니다.
- **컨트롤러 구조 및 입력 시스템 통합**: `TFDPlayerController`에서 Enhanced Input을 기반으로 어빌리티 사용과 UI 연동 제어를 관리했습니다.
- **시스템 설계 (System Design)**: `GameRuleDataAsset`을 통해 게임 규칙, 제한시간, 스폰을 데이터화하고, 멀티플레이 동기화 구조를 개선하여 서버/클라이언트 일관성을 확보했습니다.

---

## 1. 캐릭터 시스템 구축

### 1.1 공통 캐릭터 베이스 (`TFDCharacterBase`)
- 플레이어 및 AI가 공통으로 상속받는 베이스 클래스를 설계하여 GAS 기반 스탯과 스킬 로직을 통합
- `UAbilitySystemComponent`, `UTFDAttributeSet` 통합 적용  
- `PossessedBy`, `OnRep_PlayerState`에서 ASC 초기화 처리  
- `Speed`, `Gold` 속성 변화 감지 후 실시간 반영  

### 1.2 플레이어 캐릭터 (`TFDPlayerCharacter`)
- 3인칭 시점 카메라(`SpringArm`, `FollowCamera`)를 세팅하고, `GameState` 이벤트 `OnMatchInProgress`, `OnMatchWaitingPostMatch`)를 수신하여 UI와 게임 결과를 동기화하도록 구현.  

### 1.3 데이터 기반 캐릭터 설계
- 각 플레이어의 **직업(경찰 / 도둑)** 에 따라 초기 능력치, 스킬, 입력, 태그를 통합 관리하도록 구성.  
- `FTFDJobData` 구조체를 통해 스탯, 어빌리티, UI 리소스를 하나의 세트로 관리.  
- `TFDCharacterBase` 초기화 시 `PlayerDataAsset`을 참조하여 능력 자동 등록 및 태그 세팅 수행.  

---

## 2. 스킬 시스템 & UI 연동
- UTFDSkillManagerComponent`에서 `OnEffectAdded`, `OnEffectRemoved` 델리게이트를 사용해 GameplayEffect의 시작·종료 시점을 UI에 전달.  

---

## 3. 컨트롤러 및 입력 시스템

### 3.1 플레이어 컨트롤러 (`TFDPlayerController`)
- Enhanced Input 시스템을 기반으로 이동, 점프, 어빌리티를 관리.
- `JobAbility()`에서 DataAsset 기반 입력/스킬 자동 매핑  

---

## 4. 시스템 설계 (System Design)

### 4.1 게임 규칙 데이터화 (`DA_GameRuleData`)
- 제한시간, 승리 조건, 웨이브 수 등을 데이터 에셋으로 관리.  
- `TFDGameMode`가 이를 참조하여 `MatchState`를 제어하고,  
  시간이 0이 되면 자동으로 도둑 팀 승리 처리.  

### 4.2 멀티플레이 동기화 구조
- `PossessedBy()`와 `OnRep_PlayerState()` 양쪽에서 ASC 초기화(`InitAbilityActorInfo`) 수행.  
- `GameState`에서 제한시간, 팀 정보, 웨이브 상태를 Replicated 변수로 관리해  
  클라이언트 UI가 항상 최신 데이터를 표시하도록 유지.
