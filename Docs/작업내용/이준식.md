## 🛠️ 이준식

### 📋 요약
- **GAS 기반 메인 프레임워크 구축 및 테스트 어빌리티/이펙트 구현**  
- **DataTable 기반 팀 상호작용 및 스폰 시스템 설계**  
- **투사체 및 예측 이펙트 시스템 구현 (Niagara 연동)**  
- **Game Instance Subsystem 기반 사운드 관리 시스템 구축**  
- **AssetManager 기반 패키징 구조 개선 및 리소스 누락 문제 해결**

---

### 1. 메인 Gameplay Ability System (GAS) 프레임워크 구축
- **GAS 시스템 기반 프레임워크 설계 및 초기 세팅**
  - `AbilitySystemComponent`, `AttributeSet`, `GameplayAbility`, `GameplayEffect`, `AbilityTask` 등의 구조를 정립하고 전체 게임의 능력 시스템 기반을 구축  
- **테스트용 기능 구현**
  - 프레임워크 구축 후 테스트용 Gameplay Ability, Effect, Ability Task를 제작하여 시스템 연동 및 안정성 검증

---

### 2. 투사체 스킬 시스템 구현
- **GA → GA Task → Projectile 구조 설계**
  - AbilityTask 내에서 투사체 생성 및 초기화 로직 수행  
  - 투사체에 필요한 컴포넌트와 데이터를 전달하여 동적으로 생성  
- **시각 효과 및 예측 시스템**
  - 투사체에 **Niagara 이펙트**를 적용하여 발사 및 충돌 연출 강화  
  - `SweepSingleByChannel` 기반의 **예측 충돌 시스템**을 통해 시각 이펙트와 실제 충돌 타이밍을 동기화

---

### 3. 캐릭터 및 팀 상호작용 구조 설계
- **DataTable + Gameplay Tag 기반 상호작용 시스템**
  - 각 액터의 팀 정보 및 상호작용 규칙을 DataTable로 관리  
  - Gameplay Tag를 활용해 팀 간 관계(적/아군 등)를 데이터 기반으로 정의  
  - 시스템 확장성 확보를 위한 태그 중심 구조 설계  

---

### 4. 스폰 시스템 구축
#### 🔹 스폰 볼륨 (Spawn Volume)
- **위치 검증 로직**
  - Ray Trace를 활용하여 스폰 볼륨 내에서 랜덤 좌표를 선택하고,  
    지형 충돌 여부를 검사해 안전한 위치에서만 스폰 수행  
- **예외 처리**
  - 스폰된 액터가 다른 액터와 겹칠 경우 재시도 로직을 통해 안정성 확보  

#### 🔹 스폰 포인트 (Spawn Point)
- **시각적 배치**
  - Billboard 컴포넌트를 부착하여 맵 상에서 스폰 위치를 직관적으로 식별 가능  
- **데이터 연동**
  - DataTable을 통해 스폰 주기 및 아이템 목록을 관리  
  - 태그 기반으로 스폰 포인트별 고유 스폰 규칙 적용 가능  

---

### 5. 사운드 관리 시스템 구축
- **Game Instance Subsystem 기반 통합 오디오 관리**
  - BGM 및 UI 사운드를 중앙에서 제어할 수 있는 구조를 설계  
  - Game Instance에서 사운드 리소스를 관리하고, Subsystem을 통해 재생/정지 제어  
- **일관성 있는 오디오 경험 제공**
  - 전역 볼륨, BGM, UI 사운드를 통합적으로 관리하여 사용자 경험 개선  

---

### 6. 프로젝트 관리 및 패키징 개선
- **패키징 환경 문제 해결**
  - PIE(Play In Editor) 환경에서는 정상 동작했으나,  
    패키징 이후 **Nav Mesh**, **Gameplay Ability (GA, GE, DA)** 관련 리소스 누락 문제 발생  
- **해결 방안**
  - `AssetManager`의 **Primary Asset Type to Scan** 설정을 통해 필수 리소스를 패키징 대상에 명시적으로 포함  
  - **Build Path**를 지정하여 Nav Mesh를 패키징 과정에 포함  
  - **디버그 모드**를 활용하여 패키징 파일 내부 리소스 검증
